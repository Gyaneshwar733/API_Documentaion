#Introduction
SeerPortal is a web portal used for accessing all the tools and modules offered in DataSeers
The Seerporatl API is based on RESTful principle, ensuring a standardized and efficient way of interacting 
with the portal's features.
The API accepts JSON-encoded request bodies and returns JSON-encoded responses. 
It uses standard HTTP methods (like GET, POST, PUT) and response codes (like 200 for success, 404 for 
not found) to ensure consistent behaviour across all operations.
For security, all requests to the API must be made over HTTPS, protecting data during transmission and 
ensuring secure communication between the user and the server.

# Authentication
The user access is restricted with '**Token**'. Use username and password to authenticate.
Once authenticated, an API key token is generated, which is then used to access the SeerPortal API. This 
token must be included in all subsequent API requests to verify the user's identity and permissions.

# HTTP Request Methods
The HTTP request methods (also known as verbs) define the type of action the user wants to perform,
For example,
- *GET* - to **Get** data from a resource.
-  *POST* - to **Create** data at a resource.
-  *PUT* â€“ to **Update** data at a resource.
